> **测试人：** QA Engineer  
> **测试范围：** 登录功能修复验证（frontend-dev 修复登录按钮点击无响应问题）

---

## 1. 测试矩阵

| 编号 | 测试场景 | 测试类型 | 优先级 | 预期结果 |
|------|---------|---------|--------|---------|
| TC-01 | 正常登录（admin/admin123） | 正常流 | P0 | 登录成功，返回 Token，跳转 Dashboard |
| TC-02 | 错误密码（admin/wrongpassword） | 异常流 | P0 | 返回 401，显示"用户名或密码错误" |
| TC-03 | 错误用户名（nonexistent/password） | 异常流 | P0 | 返回 401，显示"用户名或密码错误" |
| TC-04 | 空表单提交 | 边界值 | P0 | 返回 422，表单验证失败 |
| TC-05 | 用户名过短（2字符） | 边界值 | P1 | 返回 422，验证失败 |
| TC-06 | 密码过短（5字符） | 边界值 | P1 | 返回 422，验证失败 |
| TC-07 | Token 刷新 | 正常流 | P0 | 使用 Refresh Token 获取新 Token |
| TC-08 | 无效 Refresh Token | 异常流 | P0 | 返回 401 |
| TC-09 | 获取当前用户信息 | 正常流 | P0 | 返回用户信息 |
| TC-10 | 未认证访问受保护接口 | 异常流 | P0 | 返回 401 |
| TC-11 | 退出登录 | 正常流 | P1 | 登出成功 |
| TC-12 | 禁用账户登录 | 异常流 | P1 | 返回 403 |
| TC-13 | 不同角色登录（teacher/student） | 正常流 | P1 | 登录成功，返回对应角色 |

---

## 2. 测试方法

### 2.1 后端 API 测试（自动化）
- 使用 pytest + FastAPI TestClient
- 测试所有 `/api/v1/auth/*` 端点
- 覆盖正常流、异常流、边界值

### 2.2 前端代码审查
- 检查 Login.vue 的 handleLogin 方法
- 检查 auth store 的 login 方法
- 检查路由守卫逻辑
- 检查 Token 刷新机制

---

## 3. 验收标准

- [ ] 所有 13 个测试用例通过
- [ ] 后端 API 测试通过率 100%
- [ ] 行覆盖率 >= 80%
- [ ] 无 JavaScript 控制台错误
- [ ] 登录按钮点击有响应（loading 状态）
- [ ] 错误提示正常显示
- [ ] Token 存储和刷新正常
- [ ] 路由守卫正常工作

## 4. 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 前端无法自动化测试 | 中 | 通过代码审查 + 手动验证 |
| Token 过期时间过长 | 低 | 手动删除 Token 模拟过期 |
