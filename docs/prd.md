## 1. 产品概述与目标

### 1.1 产品背景

在教育信息化快速发展的背景下，传统的手工记录和管理学生成绩方式已无法满足现代教学管理的需求。教师和教务人员需要一个高效、准确的成绩管理系统来提升工作效率，减少人为错误。

### 1.2 产品定义

**学生成绩管理系统**是一款面向学校教务管理人员和教师的Web应用系统，提供命令行工具和Web前端界面，用于实现学生信息的集中管理和成绩的自动化处理与统计分析。

### 1.3 目标用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| **教务管理员** | 负责学生学籍管理和成绩录入 | 学生信息维护、批量成绩录入 |
| **任课教师** | 负责特定科目的成绩管理 | 本科目成绩录入、统计分析 |
| **班主任** | 负责班级整体学业情况 | 班级成绩汇总、排名统计 |
| **学生** | 查看个人成绩和排名 | 成绩查询、个人统计 |
| **家长** | 查看学生成绩情况 | 成绩查询、学习报告 |

### 1.4 产品目标

| 目标类型 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| **效率目标** | 减少成绩处理时间 | 相比手工方式提升 80% 以上 |
| **准确目标** | 降低数据录入错误率 | 错误率 < 0.1% |
| **易用目标** | 降低学习成本 | 新用户 30 分钟内上手 |

### 1.5 核心价值

1. **数据集中化：** 统一管理学生信息和成绩数据，避免数据分散
2. **操作自动化：** 自动计算统计指标，减少人工计算
3. **查询便捷化：** 多维度快速查询，秒级响应
4. **报表智能化：** 自动生成统计报表，辅助教学决策

#### 模块一：用户认证与授权（F-AUTH）

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|
| F-AUTH-001 | 用户登录 | 🔴 P0 | 用户通过用户名密码登录系统 | 用户名、密码 | JWT Token（Access Token + Refresh Token） |
| F-AUTH-002 | Token管理 | 🔴 P0 | Token存储、刷新、过期处理 | Refresh Token | 新的Access Token |
| F-AUTH-003 | 路由守卫 | 🔴 P0 | 前端路由保护，未登录跳转登录页 | 路由访问请求 | 允许访问或重定向到登录页 |
| F-AUTH-004 | 认证状态持久化 | 🔴 P0 | 用户登录状态在页面刷新后保持 | Token | 持久化存储 |
| F-AUTH-005 | 退出登录 | 🟡 P1 | 用户主动退出系统 | 退出操作 | 清除Token，跳转登录页 |
| F-AUTH-006 | 用户信息显示 | 🟡 P1 | 在界面显示当前登录用户信息 | Token/API | 用户名、角色显示 |

**业务规则：**
- Access Token有效期：30分钟
- Refresh Token有效期：7天
- Token刷新失败时自动跳转登录页
- 登录失败显示友好错误提示
- 支持Enter键快速登录

| 功能编号 | 功能名称 | 优先级 | 功能描述 | 输入 | 输出 |
|---------|---------|--------|---------|------|------|

| F-STA-002 | 最高分统计 | 🔴 P0 | 查询指定范围的最高分 | 班级/科目/考试类型 | 最高分及学生 |
| F-STA-003 | 最低分统计 | 🔴 P0 | 查询指定范围的最低分 | 班级/科目/考试类型 | 最低分及学生 |
| F-STA-004 | 及格率统计 | 🔴 P0 | 计算及格率（≥60分） | 班级/科目/考试类型 | 及格率百分比 |

**统计指标定义：**

```
平均分 = 所有成绩总和 / 成绩数量
及格率 = (及格人数 / 总人数) × 100%
优秀率 = (优秀人数(≥90分) / 总人数) × 100%
```

## 4. 验收标准

### 4.1 功能验收标准

#### AC-001: 用户认证验收

```
场景: 用户登录
  假设 系统已启动
  当 用户输入正确的用户名和密码
  则 系统成功登录
  并且 返回JWT Token（Access Token + Refresh Token）
  并且 跳转到Dashboard页面

场景: 登录失败
  假设 用户输入错误的密码
  当 用户点击登录按钮
  则 系统拒绝登录
  并且 显示友好的错误提示"用户名或密码错误"

场景: Token自动刷新
  假设 用户已登录，Access Token即将过期
  当 用户访问受保护页面
  则 系统自动使用Refresh Token刷新
  并且 用户无感知地继续访问

场景: Token过期处理
  假设 Access Token和Refresh Token都已过期
  当 用户访问受保护页面
  则 系统跳转到登录页
  并且 显示提示"登录已过期，请重新登录"

场景: 路由守卫
  假设 用户未登录
  当 用户尝试访问受保护页面（如/dashboard）
  则 系统自动跳转到登录页
  并且 无法直接访问受保护页面

场景: 退出登录
  假设 用户已登录
  当 用户点击退出登录按钮
  则 系统清除Token
  并且 跳转到登录页
  并且 无法再访问受保护页面
```
