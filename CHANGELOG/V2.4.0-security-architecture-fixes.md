# V2.4.0 安全与架构全面优化

**发布日期**：2026-06-15

**版本类型**：安全修复 + 架构优化

---

## 修复内容

### 🔴 安全修复 (P0)

#### 1. 学生权限控制 (SEC-03)
**问题**：学生角色可通过 API 访问任意学生的成绩

**修复**：添加权限检查，学生只能查看自己的成绩

**文件**：`src/api/routes/grades.py`

#### 2. 登录接口密码泄露 (SEC-05)
**问题**：Swagger UI 文档中展示默认密码 `admin123`

**修复**：移除示例中的默认密码

**文件**：`src/schemas/auth.py`

### 🟠 架构优化 (P1)

#### 3. 成绩精度修复 (BACK-05)
**问题**：使用 `Float` 存储分数可能导致精度误差

**修复**：改用 `Numeric(4, 1)` 确保精确小数存储

**文件**：`src/models/grade.py`

#### 4. 数据库索引优化 (DB-01/02)
**问题**：缺少 `exam_date` 和 `enrollment_year` 索引

**修复**：
- Grade 模型添加 `idx_grades_exam_date` 索引
- Student 模型添加 `idx_students_enrollment_year` 索引

**文件**：`src/models/grade.py`, `src/models/student.py`

#### 5. 导入科目白名单校验 (BACK-04)
**问题**：导入成绩时不校验科目是否在允许列表中

**修复**：添加科目白名单检查，参考 `SUBJECTS` 常量

**文件**：`src/services/import_service.py`

### 🟡 代码质量 (P2)

#### 6. 文件验证逻辑抽取 (BACK-06)
**问题**：文件类型、大小验证在三个函数中重复

**修复**：抽取 `validate_upload_file()` 公共函数

**文件**：`src/api/routes/imports.py`

#### 7. 审计日志失败记录 (BACK-07)
**问题**：审计日志失败被静默吞没

**修复**：添加 `logger.warning()` 记录审计失败

**文件**：`src/api/routes/imports.py`
