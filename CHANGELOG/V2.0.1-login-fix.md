# V2.0.1 - 登录按钮修复

> **日期：** 2026-06-08  
> **类型：** Bug 修复  
> **状态：** 已完成  
> **严重程度：** 高（阻断性）

---

## 问题描述

用户打开网站后，输入用户名和密码，点击登录按钮，没有任何响应。登录功能完全不可用。

## 问题根因

响应拦截器对登录接口的 401 错误处理不当。

当用户输入错误密码时，后端返回 HTTP 401。前端 `request.ts` 的响应拦截器检测到 401 后，错误地将其当作 Token 过期处理，尝试刷新 Token。由于登录时还没有 Refresh Token，刷新必然失败，随即调用 `handleAuthError()`：

```javascript
function handleAuthError() {
  localStorage.removeItem('access_token')
  localStorage.removeItem('refresh_token')
  localStorage.removeItem('user_info')
  ElMessage.error('登录已过期，请重新登录')
  setTimeout(() => {
    window.location.href = '/login'  // 致命！100ms 后强制刷新页面
  }, 100)
}
```

`window.location.href = '/login'` 导致页面完全刷新，错误提示在 100ms 内闪现后立即消失，用户看到的就是"点击登录按钮没有任何响应"。

## 修复方案

### 修改文件

| 文件 | 修改内容 |
|------|---------|
| `frontend/src/utils/request.ts` | 让 `/auth/login` 和 `/auth/refresh` 接口的 401 错误跳过 Token 刷新逻辑，直接 reject 给调用方处理 |
| `frontend/src/stores/auth.ts` | 增强错误消息提取，兼容 FastAPI `HTTPException` 返回的 `{ detail: "..." }` 格式 |
| `frontend/src/views/login/Login.vue` | 将 `validate` 从回调风格改为 Promise 风格，增加兜底 try-catch |

### 关键代码变更

#### request.ts
```javascript
// 登录/刷新接口的 401 不走 Token 刷新逻辑
const isAuthRequest =
  originalRequest.url?.includes('/auth/login') ||
  originalRequest.url?.includes('/auth/refresh')

// 仅对非认证接口尝试 Token 刷新
if (status === 401 && !originalRequest._retry && !isAuthRequest) {
  // ... Token 刷新逻辑
}

// 认证接口的错误不在拦截器中弹消息（由 auth store 处理）
if (isAuthRequest) {
  return Promise.reject(error)
}
```

#### auth.ts
```javascript
const errData = error?.response?.data
const message = errData?.detail || errData?.error?.message || '登录失败，请检查用户名和密码'
```

#### Login.vue
```javascript
// 修改前：回调风格
await formRef.value.validate(async (valid) => { ... })

// 修改后：Promise 风格
const valid = await formRef.value.validate()
```
