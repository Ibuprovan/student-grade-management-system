## 更新内容

### 1. 修复生产环境 JWT 密钥检查导致启动失败的问题

**问题描述**：
- 当 `DEBUG=False`（默认值）时，系统判定为生产环境
- 生产环境强制要求设置自定义 `JWT_SECRET_KEY`
- 新克隆项目未配置 `.env` 文件时，后端无法启动

**解决方案**：
- 新增 `.env.example` 配置模板文件
- 在 README 中添加环境配置说明
- 引导开发者首次运行前正确配置环境变量

### 2. 新增环境变量配置模板

**新增文件**：`.env.example`

包含所有可配置的环境变量及详细注释：
- `DEBUG`：调试模式开关
- `DATABASE_URL`：数据库连接字符串
- `JWT_SECRET_KEY`：JWT 签名密钥
- `CORS_ORIGINS`：跨域资源共享配置
- 数据库连接池、分页等高级配置

### 3. 完善项目文档

**更新文件**：`README.md`

- 新增「环境配置」章节，位于「快速开始」首位
- 添加关键配置变量说明表格
- 添加 JWT 密钥生成命令
- 更新项目结构，添加 `.env.example` 文件说明

## 升级指南

### 新项目

```bash
# 1. 克隆项目
git clone <repository-url>

# 2. 复制配置模板
cp .env.example .env

# 3. 按需修改配置（开发环境可直接使用默认值）

# 4. 启动项目
```

### 已有项目

如果项目已有 `.env` 文件且运行正常，无需任何改动。

如果遇到 `RuntimeError: 生产环境禁止使用默认 JWT 密钥` 错误：

```bash
# 方案一：开启调试模式（开发环境推荐）
echo DEBUG=True >> .env

# 方案二：设置自定义密钥（生产环境必须）
python -c "import secrets; print(secrets.token_urlsafe(32))"
# 将生成的密钥写入 .env
echo JWT_SECRET_KEY=<生成的密钥> >> .env
```
