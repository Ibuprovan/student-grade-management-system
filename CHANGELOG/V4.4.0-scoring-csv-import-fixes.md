# V4.4.0 - 成绩计算验证优化与导入修复

> **发布日期：** 2026-06-18
> **版本类型：** Bug 修复 + 功能优化
> **主要变更：** 成绩计算逻辑修复、CSV导入增强、批量删除功能、表格显示优化、GradeList 重构

---

## 更新摘要

本次更新聚焦于成绩计算的准确性、导入流程的健壮性以及 UI 显示的稳定性。全面修复了及格率/优秀率计算逻辑、语数英满分验证、CSV 导入兼容性，新增批量删除功能，并重构了成绩列表页面。

---

## 新增功能

### 1. 批量删除成绩与学生
- **一键删除全部成绩**：删除全部或按班级筛选删除
- **一键删除全部学生**：同时清理关联成绩
- **确认弹窗**：危险操作前弹出确认对话框
- **接口优化**：删除全部接口改为 POST 请求，避免 422 错误

### 2. 总分独立存储
- **total_score 字段**：grades 表新增 total_score，成绩录入时自动汇总各科分数存储
- **统计概览改进**：直接从 total_score 聚合统计（后因兼容性回退到各科聚合兜底）
- **总分排名优化**：显示全部学生，移除前 10 名限制

### 3. GradeList 重构
- **移除成绩明细 tab**：只保留总分排名视图，页面更加简洁
- **搜索栏驱动排名**：班级/考试类型筛选直接触发排名查询
- **关键字实时过滤**：学号/姓名输入框实时过滤当前排名数据

## Bug 修复

### 成绩计算与验证
| 问题 | 修复 |
|------|------|
| 及格率/优秀率按实际科目数计算不准确 | 改为按 750 满分统一计算及格/优秀阈值 |
| 单科及格率/优秀率仍按 750 满分计算 | 改为按科目各自满分（主科 150 / 其他 100）分别计算 |
| 语数英成绩无法录入超过 100 分 | 全面放开满分限制：录入、导入、批量更新均支持 150 分上限 |
| 统计概览总分与各科聚合不一致 | 从独立 total_score 字段改为从各科成绩实时聚合，确保一致性 |
| 班级统计不基于总分 | 班级统计全部改为基于总分维度进行聚合计算 |

### CSV 导入修复
| 问题 | 修复 |
|------|------|
| CSV 文件包含 BOM 字符导致学号匹配失败 | 自动检测并移除 UTF-8 BOM 头 |
| 前端 CSV 导入学号字段名不匹配 | 统一为 `学号` / `student_id` 双字段兼容 |
| 导入预览表格列名与模板不一致 | 列名按科目分组动态生成，与模板列顺序一致 |
| 导入服务分数验证不区分科目 | 导入时分数验证改为科目感知，主科 150 / 其他 100 |

### 错误处理增强
| 问题 | 修复 |
|------|------|
| 成绩响应序列化未捕获异常 | 显式捕获序列化异常并返回友好错误信息 |
| 成绩搜索端点无错误日志 | 添加完整 try-catch 和日志记录 |

### 表格显示修复
| 问题 | 修复 |
|------|------|
| 页面容器 height:100% 导致表格行被截断 | 移除 height:100%，改用内容自然撑开 |
| 布局 flex + overflow 限制剪裁内容 | 移除多余 flex 高度约束和 overflow 限制 |
| html/body height:100% 导致布局异常 | 改用 min-height:100vh 替代 height:100% |
| el-table 内部 overflow:hidden 导致行高不完整 | 移除 el-table 上的 overflow:hidden 样式 |
