# V2.3.1 批量导入功能全面修复

**发布日期**：2026-06-14

**版本类型**：Bug 修复

---

## 修复内容

### 1. 修复成绩批量导入 404 错误

**问题**：前端请求 `/api/v1/import/grades` 返回 404

**原因**：后端路由前缀缺少 `/api/v1`

**修复**：`imports.py` 路由前缀从 `/import` 改为 `/api/v1/import`

### 2. 修复 FormData 上传失败 (422 错误)

**问题**：上传文件时返回 422 Unprocessable Entity

**原因**：axios 实例默认 `Content-Type: application/json` 覆盖了 FormData 的 `multipart/form-data`

**修复**：
- `request.ts` 的 `upload` 函数设置 `Content-Type: undefined`，让浏览器自动设置 boundary
- `grade.ts` 改用 `upload` 函数替代 `post` 函数

### 3. 修复 CSV 文件 BOM 编码问题

**问题**：导入 CSV 时提示"学号为空"

**原因**：CSV 文件开头有 UTF-8 BOM 字符 (`\ufeff`)，导致表头第一个字段变成 `﻿学号`

**修复**：
- 使用 `utf-8-sig` 编码优先解析
- 清理字段名空白字符
- 去除可能残留的 BOM 字符

### 4. 修复成绩导入数据库类型错误

**问题**：SQLite Date type only accepts Python date objects

**原因**：`exam_date` 以字符串形式传入数据库，需要 date 对象

**修复**：`_validate_grade_row` 方法将 `exam_date` 字符串转换为 `datetime.date` 对象

### 5. 修复导入结果显示 undefined

**问题**：导入成功后显示"成功0条，失败0条，共计0条"

**原因**：
- 前端使用 `importResult?.total` 但后端返回 `total_rows`
- store 未正确提取 `response.data`

**修复**：
- `GradeImport.vue` 改为 `importResult?.total_rows`
- `stores/grade.ts` 添加 `(response as any).data || response`
- `types/grade.ts` 更新 `BatchImportResponse` 类型定义

### 6. 修复分页组件废弃警告

**问题**：Element Plus 提示 `small` 属性即将废弃

**修复**：`el-pagination` 的 `small` 属性改为 `size="small"`

### 7. 新增成绩批量导入后端接口

**新增接口**：
- `POST /api/v1/import/grades` - 批量导入成绩
- `GET /api/v1/import/grades/template` - 下载成绩导入模板

**功能特性**：
- 支持 CSV/Excel 格式
- 自动检测学生是否存在
- 支持更新已有成绩（唯一约束：学号+科目+考试类型）
- 事务处理保证数据一致性
