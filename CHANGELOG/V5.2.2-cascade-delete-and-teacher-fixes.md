# V5.2.2 - 删除学生级联删除账号与教师相关修复

> **发布日期：** 2026-06-19
> **版本类型：** Bug 修复 + 功能优化
> **主要变更：** 删除学生时同步删除关联用户账号、修复教师登录/仪表盘问题、修复 Login.vue 编译错误

---

## 更新摘要

本次更新修复了管理员端删除学生时未同步删除关联用户账号的问题，同时修复了教师角色上线后遗留的若干问题（登录跳转、仪表盘卡片布局、模板编译错误），并清理了数据库中残留的 teacher/student 默认账号。

---

## Bug 修复

| 问题 | 修复 |
|------|------|
| 管理员端删除学生后，`users` 表对应账号仍残留 | `delete_student()` 新增级联删除 `users` 表关联用户 |
| 教师登录后被重定向到管理员仪表盘 `/dashboard` | 路由守卫中添加 teacher 角色跳转到 `/t/dashboard` |
| 教师仪表盘 / 统计概览卡片未填满整行 | 改用 `flex` 布局+百分比宽度覆盖全局限制 |
| `Login.vue` 模板 `</div>` 标签未闭合导致 Vite 编译错误 | 补全缺失的闭合标签 |

## 清理

### 删除数据库中的 teacher/student 默认账号
- V5.2.1 已从 `init_users.py` 移除代码配置，本次实际删除数据库中对应记录
- 执行 `DELETE FROM users WHERE username IN ('teacher', 'student')`

### 级联删除逻辑
- `StudentService.delete_student()` 在删除学生记录前，先通过 `UserRepository.get_by_username()` 查找关联用户并删除
- 完整的删除流程：`用户表删除 → 学生表删除 (ORM cascade 触发成绩级联删除)`
