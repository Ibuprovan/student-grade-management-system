# V5.2.4 - 学生详情页重构与统计服务层全面修复

> **发布日期：** 2026-06-19
> **版本类型：** Bug 修复 + 功能优化
> **主要变更：** 学生详情页成绩记录重构、统计服务层考试类型混合问题全面修复

---

## 更新摘要

本次更新包含两部分：一、对学生详情页的成绩记录板块进行重构，支持考试类型筛选和分页加载；二、对统计服务层进行深度修复，确保所有统计接口在未指定考试类型时自动取最新考试类型，彻底消除跨考试类型混合统计问题。

---

## Bug 修复

### 学生详情页
| 问题 | 修复 |
|------|------|
| 成绩记录一次性加载全部数据，页面卡顿 | 改为每页 20 条分页加载 |
| 移动端卡片视图与表格视图重复 | 移除卡片视图，只保留表格 |
| 表格列使用固定宽度，右侧留白严重 | 改为 `min-width` 自适应占满整页 |
| 不支持按考试类型筛选 | 卡片标题栏新增考试类型下拉筛选器 |
| 成绩汇总卡片计算错误（混合所有考试类型） | 改用 `/statistics/student` API 获取准确数据 |
| `getScoreTagType` 返回空字符串导致 el-tag 报错 | 80-89 分返回 `'primary'` 而非空字符串 |
| `fetchSummary` 使用 `page_size=500` 超后端限制 | 改用统计 API，不再需要大分页请求 |

### 统计服务层
| 问题 | 修复 |
|------|------|
| `_get_scores()` 跨考试类型混合统计 | 未指定 `exam_type` 时默认取最新考试类型 |
| `get_batch_subject_statistics()` 跨考试类型混合 | 未指定 `exam_type` 时默认取最新考试类型 |
| `get_report()` 返回值中 `exam_type` 为 None | 调用前先解析实际使用的考试类型 |
| `get_statistics_metrics()` 同上 | 同上 |
