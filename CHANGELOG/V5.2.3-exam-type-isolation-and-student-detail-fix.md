# V5.2.3 - 统计信息考试类型隔离与学生详情修复

> **发布日期：** 2026-06-19
> **版本类型：** Bug 修复
> **主要变更：** 修复多种考试类型下统计数据混合计算、学生详情页加载失败

---

## 更新摘要

本次更新修复了系统中存在已久的两个问题：一、当录入多种考试类型（如期中、期末）后，仪表盘和统计页面将不同考试类型的成绩混合计算，导致平均分、及格率、总分等数据严重失真；二、管理员/教师在学生列表点击查看学生详情时，页面一直加载无法显示。

---

## Bug 修复

### 统计信息考试类型混合
| 问题 | 修复 |
|------|------|
| 班主任仪表盘平均分/及格率跨考试类型混合统计 | 新增 `exam_type` 参数，前端添加考试类型下拉筛选 |
| 学科组长仪表盘同上 | 新增 `exam_type` 参数，前端添加考试类型下拉筛选 |
| 教师仪表盘同上 | 前端添加考试类型下拉筛选（后端已有参数） |
| 统计概览 grade_count 未按考试类型过滤 | 添加 `exam_type` 过滤条件 |
| `_get_scores()` 跨考试类型混合统计 | 未指定 `exam_type` 时默认取最新考试类型（影响所有使用该方法的统计接口） |
| `get_batch_subject_statistics` 跨考试类型混合统计 | 未指定 `exam_type` 时默认取最新考试类型 |
| `get_batch_class_statistics` 跨考试类型求和 | 未指定 `exam_type` 时默认取最新考试类型 |
| `get_student_statistics` 跨考试类型求和 | 未指定 `exam_type` 时默认取最新考试类型 |
| `get_total_score_report` 跨考试类型求和 | 未指定 `exam_type` 时默认取最新考试类型 |

### 学生详情加载失败
| 问题 | 修复 |
|------|------|
| `/grades/search` 不接受 `student_id` 参数 | 新增 `student_id` 可选查询参数 |
| `StudentDetail.vue` 响应解析错误 `gradeResponse.items` | 修正为 `gradeResponse.data?.items` |
| 学生详情路由仅允许 admin/teacher | 开放给 `class_teacher`、`subject_leader` 角色 |

> **发布说明：** 本次更新解决了多考试类型场景下统计数据失真的核心问题。统计服务层 `_get_scores()` 方法（被平均分、最高分、最低分、及格率、优秀率、综合报告等接口共用）现在未指定考试类型时自动取最新考试类型，彻底消除跨考试类型混合统计。所有仪表盘和统计页面支持按考试类型筛选。
