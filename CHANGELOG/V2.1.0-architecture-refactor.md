## 更新概述

基于架构分析发现的 25 个摩擦点，完成了 10 项架构优化，消除代码重复、统一依赖管理、提升可维护性。

## 前端架构优化

### 1. 统一 Token 刷新机制
- **问题**：Token 刷新在 `auth.ts` Store 和 `request.ts` 拦截器中双重实现
- **修复**：删除 Store 中的重复实现，统一由拦截器处理
- **文件**：`stores/auth.ts`、`utils/request.ts`

### 2. 抽取通用 CSV 导出工具函数
- **问题**：`useStudent.ts` 和 `useGrade.ts` 中各自实现了相同的 CSV 导出逻辑
- **修复**：新建 `utils/export.ts`，提供 `downloadCSV` 函数
- **文件**：`utils/export.ts`（新建）、`composables/useStudent.ts`、`composables/useGrade.ts`

### 3. 统一前端常量定义
- **问题**：PASS_SCORE、EXCELLENT_SCORE 等分数线在多处硬编码
- **修复**：在 `utils/format.ts` 中定义 `SCORE_THRESHOLDS` 常量
- **文件**：`utils/format.ts`

### 4. 删除未使用的 usePagination
- **问题**：`usePagination.ts` 从未被使用
- **修复**：删除文件
- **文件**：`composables/usePagination.ts`（删除）

### 5. 修复 GradeForm.vue 绕过 Store 直接调用 API
- **问题**：`GradeForm.vue` 直接导入底层 API 函数
- **修复**：将调用封装到 `useGradeForm` composable 中
- **文件**：`composables/useGrade.ts`、`views/grade/GradeForm.vue`

## 后端架构优化

### 6. 统一常量引用
- **问题**：`PASS_SCORE` 和 `EXCELLENT_SCORE` 在 3 处各自定义
- **修复**：统一从 `constants.py` 引用
- **文件**：`services/statistics_service.py`、`services/dashboard_service.py`

### 7. 统一依赖注入
- **问题**：`DashboardService` 的依赖注入定义在路由文件中
- **修复**：统一移到 `dependencies.py`
- **文件**：`api/dependencies.py`、`api/routes/dashboard.py`

### 8. 抽取分页响应构建函数
- **问题**：分页响应构建逻辑在 students.py 和 grades.py 中重复 5 次
- **修复**：创建 `build_paginated_response` 辅助函数
- **文件**：`core/utils.py`（新建）、`api/routes/students.py`、`api/routes/grades.py`

### 9. 抽取公共验证逻辑
- **问题**：`validate_score` 在 3 个 Schema 类中重复
- **修复**：创建 `_validate_score_precision` 公共函数
- **文件**：`schemas/grade.py`

### 10. 创建 UserRepository
- **问题**：auth.py 路由直接使用 `db.query(User)` 绕过 Repository 模式
- **修复**：创建 `UserRepository`，统一用户数据访问
- **文件**：`repositories/user_repo.py`（新建）、`api/routes/auth.py`
