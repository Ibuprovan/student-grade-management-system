## 1. 系统概述

学生成绩管理系统是一个全栈 Web 应用，采用**专业教育风格浅色主题**，支持**五级角色权限**：

| 角色 | 权限范围 |
|------|----------|
| 管理员 | 全部权限：学生/成绩管理、统计分析、班主任/学科组长分配、账号管理 |
| 教师 | 学生/成绩管理、统计分析（可增删改） |
| 班主任 | 仅查看本班学生/成绩/统计（只读） |
| 学科组长 | 仅查看负责学科的全年级成绩与统计（只读） |
| 学生 | 仅查看自己的成绩 |

## 2. 安装与启动

### 2.1 环境准备

| 软件 | 版本要求 |
|------|----------|
| Python | 3.9 或更高版本 |
| Node.js | 16 或更高版本 |
| 浏览器 | Chrome / Firefox / Edge（最新版） |

### 2.2 下载项目

```bash
git clone https://github.com/Ibuprovan/student-grade-management-system.git
cd student-grade-management-system
```

### 2.3 一键启动（推荐，仅 Windows）

双击 `start.bat`，脚本会自动完成：
1. 安装 Python 后端依赖
2. 安装前端 Node.js 依赖
3. 初始化数据库（创建默认管理员账号）
4. 启动后端服务（`http://localhost:8000`）
5. 启动前端界面（`http://localhost:5173`）

> **注意**：启动后不要关闭弹出的命令行窗口，否则服务会停止。

### 2.4 手动启动

**终端 1 — 后端：**

```bash
pip install -r requirements.txt
python -m src.scripts.init_users
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

**终端 2 — 前端：**

```bash
cd frontend
npm install
npm run dev
```

### 2.5 验证启动

打开浏览器访问 **http://localhost:5173**，看到登录页面即表示启动成功。

## 3. 登录与账号管理

### 3.1 登录

1. 打开浏览器访问 `http://localhost:5173`
2. 输入用户名和密码
3. 点击「登录」

### 3.2 首次登录强制改密

以下情况需要先修改密码才能进入系统：
- **班主任**首次登录（初始密码 `123456`）
- **学科组长**首次登录（初始密码 `123456`）
- 管理员在账号管理中重置密码后
- 系统新创建的学生账号

系统会自动弹出修改密码对话框，按要求输入：
- 旧密码
- 新密码（至少 8 位）
- 确认新密码

### 3.3 修改密码

登录后点击右上角头像 → 「修改密码」，可随时修改自己的密码。

### 3.4 账号类型说明

| 角色 | 用户名示例 | 密码 | 如何获得 |
|------|-----------|------|----------|
| 管理员 | `admin` | `admin123` | 系统默认 |
| 教师 | （不再使用） | — | 已弃用 |
| 班主任 | `2026001`（入学年份 + 班级编号） | `123456`（强制改密） | 管理员在后台分配 |
| 学科组长 | `chinese` / `math` / `english` ... | `123456`（强制改密） | 管理员在后台分配 |
| 学生 | `20260001`（学号） | `123456` | 系统自动生成 |

## 4. 管理员操作

管理员登录后看到完整的侧边栏菜单。

### 4.1 仪表盘

- 查看系统总览：学生总数、成绩记录数、平均分、及格率
- 快捷操作入口（一键跳转到常用功能）

### 4.2 学生管理

**学生列表：**
- 搜索：按学号或姓名搜索
- 筛选：按班级下拉选择
- 排序：点击列标题可排序
- 分页：每页可切换 10/20/50 条
- 操作：点击「编辑」修改学生信息，点击「删除」移除

**添加学生：**
1. 点击左侧「学生管理 → 添加学生」
2. 填写学号（8 位数字，如 `20260001`）、姓名、性别、入学年份
3. 班级从下拉列表中选择（格式：`2026级1班`）
4. 点击「保存」

**批量导入学生：**
1. 点击「批量导入」
2. 下载 Excel/CSV 模板
3. 按模板格式填写学生数据
4. 上传文件，系统自动校验并导入
5. 查看导入结果（成功数、失败数、失败原因）

**导出数据：** 在学生列表页点击「导出」按钮，下载 CSV 文件。

### 4.3 成绩管理

**成绩录入（一次性录入 9 科）：**
1. 点击「成绩管理 → 成绩录入」
2. 选择考试类型（如「期中考试」）和考试日期
3. 搜索学生（输入学号或姓名）
4. 在表格中填写各科成绩：
   - 语文/数学/英语：满分 150 分
   - 物理/化学/生物/政治/历史/地理：满分 100 分
5. 系统自动计算总分
6. 点击「提交」保存，或「提交并继续」保存后录入下一位学生

**成绩列表：**
- 总分排名视图（默认）：显示每位学生的总分及排名
- 支持按班级、科目、考试类型、学号/姓名多维度筛选
- 支持按总分、学号排序
- 点击「导出」下载数据

**批量导入成绩：**
1. 点击「成绩导入」
2. 下载模板（每人一行：学号 + 姓名 + 9 科成绩 + 考试类型 + 日期）
3. 填写后上传，系统自动解析并导入

### 4.4 统计分析

**统计概览：**
- 统计卡片：学生总数、平均总分、最高总分、及格率（≥60）、优秀率（≥90）
- 总分分布柱状图：展示各分数段人数分布
- 科目趋势折线图：对比各科目历次考试平均分趋势
- 科目占比饼图：各科目平均分占比
- 三维能力雷达图：平均分 + 及格率 + 优秀率
- 总分排名表：所有学生排名，可查看各科明细
- 单科排名表：可选科目查看单科排名

**班级统计：**
- 选择「班级」和「考试类型」后查询
- 概览卡片：该班级学生人数、平均分、及格率
- 班级统计表格：按 `2026级1班 → 2班 → 3班 ...` 顺序排列
- 柱状图：班级平均分对比
- 折线图：班级及格率/优秀率对比

**科目统计：**
- 选择「科目」和「考试类型」后查询
- 概览卡片：最高分、最低分、平均分、及格率、优秀率
- 科目对比柱状图
- 及格率/优秀率折线图
- 分数分布柱状图（区分 150 满分和 100 满分科目）
- 能力雷达图

### 4.5 班主任管理

**用途：** 为班级分配班主任，系统自动创建班主任账号。

**操作步骤：**
1. 点击侧边栏「班主任管理」
2. 点击「添加班主任」
3. 在弹出的对话框中选择班级（只显示未分配的班级）
4. 系统自动生成班主任姓名（与班级同名）
5. 点击「确认」，系统自动：
   - 创建用户账号（用户名 = `入学年份 + 班级编号`，如 `2026001`）
   - 密码设为 `123456`（首次登录强制改密）
   - 自动生成该班级所有学生的成绩汇总数据
6. 删除班主任：列表中选择「删除」，同时删除关联的用户账号

**查看列表：** 显示所有班主任及负责班级，支持搜索。

### 4.6 学科组长管理

**用途：** 为学科分配组长，系统自动创建学科组长账号。

**操作步骤：**
1. 点击侧边栏「学科组长管理」
2. 点击「添加学科组长」
3. 选择学科（语文/数学/英语/物理/化学/生物/政治/历史/地理）
4. 填写组长姓名
5. 点击「确认」，系统自动：
   - 创建用户账号（用户名 = 学科英文名小写，如 `chinese`）
   - 密码设为 `123456`（首次登录强制改密）
6. 删除学科组长：列表中选择「删除」，同时删除关联用户账号

**注意：** 同一学科只能有一位组长，已分配的学科不会出现在下拉列表中。

### 4.7 账号管理

**用途：** 统一查看所有账号信息，一键重置密码。

访问侧边栏「账号管理」，包含三个子页面：

**学生账号：**
- 查看所有学生账号：学号、姓名、班级、启用状态、是否需改密
- 点击「重置密码」：密码重置为 `123456`，标记为需改密

**班主任账号：**
- 查看所有班主任账号：用户名、班主任姓名、负责班级、入学年份
- 点击「重置密码」

**学科组长账号：**
- 查看所有学科组长账号：用户名、学科（中英文）、组长姓名
- 点击「重置密码」

## 5. 教师操作

教师登录后的侧边栏与管理员大致相同，但**没有**管理类菜单项：
- 无「班主任管理」
- 无「学科组长管理」
- 无「账号管理」

**可操作功能：**
- 仪表盘
- 学生管理（增删改查 + 批量导入 + 导出）
- 成绩管理（录入、列表、批量导入 + 导出）
- 统计分析（概览、班级统计、科目统计）

> 教师拥有完整的数据增删改权限。

## 6. 班主任操作

班主任登录后看到**班主任专属菜单**，所有页面**只读**，仅展示本班数据。

### 6.1 班级仪表盘

- 统计卡片：本班学生人数、成绩记录数、平均总分
- 本班概况数据

### 6.2 学生信息

- 查看本班学生列表
- 支持搜索、筛选、分页
- **不能**添加、编辑或删除学生

### 6.3 成绩信息

- 查看本班学生成绩列表
- 总分排名视图：总分 + 各科成绩 + 排名
- 支持按科目、考试类型、日期筛选
- **不能**录入或修改成绩

### 6.4 统计分析

**统计概览：**
- 本班统计卡片：平均总分、及格人数、优秀人数
- 总分分布柱状图
- 总分排名表

**科目统计：**
- 选择科目查看
- 分数分布柱状图（区分 150/100 满分）
- **不能**查看其他班级数据

## 7. 学科组长操作

学科组长登录后看到**学科组长专属菜单**，所有页面**只读**，仅展示该学科的全年级数据。

### 7.1 学科仪表盘

- 统计卡片：该学科学生总数、成绩记录数、平均分、最高分、最低分
- 全年级概况数据

### 7.2 成绩管理

- 查看全年级该学科的所有成绩
- 支持按班级筛选（加载全部班级，不限制当前页）
- 支持按考试类型筛选
- 显示各班级的学科成绩数据

### 7.3 统计概览

- **班级分数对比图**：折线图展示各班级的平均分/最高分/最低分
- 支持勾选显示维度：☑ 平均分、☑ 最高分、☑ 最低分
- 可同时选择多个维度叠加显示

## 8. 学生操作

学生登录后只看到一个菜单项「我的成绩」。

### 我的成绩

- **成绩概览卡片**：
  - 总成绩（保留 1 位小数）
  - 通过人数（≥60 分科目数）
  - 优秀人数（≥90 分科目数）
  - "≥60 为及格，≥90 为优秀" 提示
- **成绩明细表**：列出所有科目成绩，表格全宽显示
- 支持按考试类型筛选查看

## 9. 数据导入导出

### 9.1 导入学生

**模板格式：**

| 学号 | 姓名 | 性别 | 班级 | 入学年份 |
|------|------|------|------|----------|
| 20260001 | 张三 | 男 | 2026级1班 | 2026 |
| 20260002 | 李四 | 女 | 2026级1班 | 2026 |

- 支持 `.xlsx` / `.xls` / `.csv` 格式
- 学号必须唯一
- 班级格式必须为 `2026级X班`

### 9.2 导入成绩

**模板格式：**

| 学号 | 姓名 | 语文 | 数学 | 英语 | 物理 | 化学 | 生物 | 政治 | 历史 | 地理 | 考试类型 | 考试日期 |
|------|------|------|------|------|------|------|------|------|------|------|----------|----------|
| 20260001 | 张三 | 135 | 128 | 140 | 92 | 88 | 85 | 78 | 90 | 82 | 期中考试 | 2026-06-15 |

**注意事项：**
- 学号必须已存在于学生列表中
- 语文/数学/英语：0–150 分
- 其他科目：0–100 分
- 总分由系统自动计算
- 如已有该学生该考试类型的成绩，将自动覆盖

### 9.3 导出数据

学生列表和成绩列表页面均提供「导出」按钮，点击后生成 CSV 文件自动下载。

## 10. 常见问题

### Q1: 忘记密码怎么办？
联系管理员。管理员在「账号管理」中找到你的账号，点击「重置密码」将密码重置为 `123456`。下次登录时需修改密码。

### Q2: 为什么无法登录？
- 检查用户名和密码是否输入正确
- 确认后端服务是否正在运行（访问 `http://localhost:8000/docs` 测试）
- 首次登录若提示需要改密，请按提示设置新密码

### Q3: 添加班主任时班级下拉为空？
说明所有班级都已分配了班主任。可以先删除某位班主任再重新分配，或者该班级已有班主任。

### Q4: 添加学科组长时学科下拉为空？
说明所有学科都已分配了组长。同一学科只能有一位组长。

### Q5: 班主任/学科组长能看到其他班的数据吗？
不能。班主任只能看到自己负责班级的数据；学科组长只能看到自己负责学科的全年级数据。都是只读权限。

### Q6: 数据保存在哪里？
所有数据保存在项目根目录的 `data/grades.db` 文件中（SQLite 数据库）。**请定期备份此文件。**

### Q7: 如何备份数据？
直接将 `data/grades.db` 复制到安全位置保存即可。恢复时将备份文件覆盖回原位置。

### Q8: 如何修改端口？
- 后端：修改启动命令中的 `--port` 参数
- 前端：编辑 `frontend/vite.config.ts` 中的 `server.port`

### Q9: 启动报错 "端口被占用"？
关闭占用该端口的程序，或修改端口。Windows 上可使用 `netstat -ano | findstr :8000` 查看占用进程 PID，在任务管理器中结束该进程。

### Q10: 如何更新系统？
```bash
# 1. 备份数据库
cp data/grades.db data/grades.db.bak

# 2. 拉取最新代码
git pull

# 3. 更新依赖
pip install -r requirements.txt
cd frontend && npm install && cd ..

# 4. 重启服务
# （关闭旧窗口，重新运行 start.bat 或手动启动）
```
