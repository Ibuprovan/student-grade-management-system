## 1. 系统概述

学生成绩管理系统是一个面向学校的全栈 Web 应用，支持五级角色权限管理。

### 1.1 角色与权限

| 角色 | 权限范围 | 数据范围 |
|------|----------|----------|
| 管理员 | 全部功能：增删改查、统计分析、角色分配、账号管理 | 全校数据 |
| 教师 | 学生管理、成绩管理、统计分析（可增删改） | 所分配科目+班级的数据 |
| 班主任 | 查看本班学生、成绩、统计（只读） | 本班数据 |
| 学科组长 | 查看负责学科的全年级成绩与统计（只读） | 该学科全年级数据 |
| 学生 | 查看自己的成绩（只读） | 本人数据 |

### 1.2 功能模块

- **学生管理**：学生信息的增删改查、批量导入导出
- **成绩管理**：成绩录入、批量导入、成绩列表、导出
- **统计分析**：统计概览、班级统计、科目统计、总分排名、单科排名
- **角色管理**：班主任分配、学科组长分配、教师任课分配
- **账号管理**：统一查看所有账号、一键重置密码

### 1.3 科目与考试类型

**9 个科目：** 语文、数学、英语（满分 150）、物理、化学、生物、政治、历史、地理（满分 100）

**4 种考试类型：** 期中、期末、月考、单元测试

## 2. 安装与启动

### 2.1 环境要求

| 软件 | 最低版本 |
|------|----------|
| Python | 3.9+ |
| Node.js | 16+ |
| 浏览器 | Chrome / Firefox / Edge 最新版 |

### 2.2 下载项目

```bash
git clone https://github.com/Ibuprovan/student-grade-management-system.git
cd student-grade-management-system
```

### 2.3 一键启动（Windows 推荐）

双击项目根目录下的 `start.bat`，脚本自动完成：
1. 安装 Python 后端依赖
2. 安装前端 Node.js 依赖
3. 初始化数据库（创建默认管理员账号 `admin` / `admin123`）
4. 启动后端（`http://localhost:8000`）
5. 启动前端（`http://localhost:5173`）

> 启动后不要关闭弹出的命令行窗口，否则服务停止。

### 2.4 手动启动

**终端 1 — 后端：**
```bash
pip install -r requirements.txt
python -m src.scripts.init_users
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

**终端 2 — 前端：**
```bash
cd frontend
npm install
npm run dev
```

### 2.5 验证

打开浏览器访问 `http://localhost:5173`，看到登录页面即启动成功。

后端 API 文档：`http://localhost:8000/docs`（Swagger UI）

### 2.6 配置文件

复制 `.env.example` 为 `.env`，可修改 JWT 密钥等配置。默认配置无需修改即可使用。

## 3. 登录与账号体系

### 3.1 登录流程

1. 打开 `http://localhost:5173`
2. 输入用户名和密码
3. 点击「登录」
4. 系统根据角色自动跳转到对应的首页

### 3.2 首次登录强制改密

以下账号首次登录时，系统会弹出修改密码对话框，必须先设置新密码才能进入：
- 班主任账号（初始密码 `123456`）
- 学科组长账号（初始密码 `123456`）
- 教师账号（初始密码 `123456`）
- 管理员重置密码后的任何账号

新密码要求：至少 8 位，不能与旧密码相同。

### 3.3 账号命名规则

| 角色 | 用户名格式 | 示例 | 初始密码 |
|------|-----------|------|----------|
| 管理员 | 固定 | `admin` | `admin123` |
| 班主任 | `{入学年份}{班级编号}` | `2026001` | `123456` |
| 学科组长 | 学科英文名小写 | `chinese`、`math` | `123456` |
| 教师 | `{学科英文}{入学年份}{班级编号}` | `Chinese2026001` | `123456` |
| 学生 | 学号 | `20260001` | `123456` |

### 3.4 修改密码

登录后点击右上角头像 → 「修改密码」，输入旧密码和新密码即可。

## 4. 管理员操作指南

管理员登录后看到完整侧边栏菜单，拥有系统全部权限。

### 4.1 仪表盘

**路径：** 侧边栏 → 仪表盘

显示系统总览：
- 学生总数、成绩记录数、平均分、及格率
- 快捷操作入口

### 4.2 学生管理

**路径：** 侧边栏 → 学生管理

#### 4.2.1 学生列表

- **搜索**：输入学号或姓名关键词
- **班级筛选**：下拉选择班级
- **排序**：点击列标题排序
- **分页**：底部切换每页条数（10/20/50）
- **查看详情**：点击「查看」进入学生详情页
- **编辑**：点击「编辑」修改学生信息
- **删除**：点击「删除」移除学生（同时删除关联账号和成绩）

#### 4.2.2 添加学生

1. 点击侧边栏「学生管理 → 添加学生」
2. 填写信息：
   - 学号：8 位数字（如 `20260001`）
   - 姓名、性别
   - 班级：从下拉选择（格式 `2026级1班`）
   - 入学年份
3. 点击「保存」

> 添加学生后，系统自动创建对应的登录账号（用户名 = 学号，密码 `123456`）。

#### 4.2.3 批量导入学生

1. 点击「批量导入」
2. 下载 Excel/CSV 模板
3. 按模板格式填写（学号、姓名、性别、班级、入学年份）
4. 上传文件
5. 系统显示导入结果：成功数、失败数、失败原因

#### 4.2.4 导出学生

在学生列表页点击「导出」，下载 CSV 文件。

### 4.3 成绩管理

**路径：** 侧边栏 → 成绩管理

#### 4.3.1 成绩录入

1. 点击「成绩管理 → 成绩录入」
2. 选择考试类型（期中/期末/月考/单元测试）和考试日期
3. 搜索学生（输入学号或姓名）
4. 在表格中填写各科成绩：
   - 语文/数学/英语：0–150 分
   - 物理/化学/生物/政治/历史/地理：0–100 分
5. 系统自动计算总分
6. 点击「提交」保存，或「提交并继续」保存后录入下一位学生

#### 4.3.2 成绩列表

- 默认显示总分排名视图（每人一行，各科展开为列）
- 支持按班级、科目、考试类型、学号/姓名筛选
- 支持按总分、学号排序
- 点击「导出」下载数据

#### 4.3.3 批量导入成绩

1. 点击「成绩导入」
2. 下载模板（每人一行：学号 + 姓名 + 9 科成绩 + 考试类型 + 日期）
3. 填写后上传
4. 系统自动解析并导入，显示结果

> 如已有该学生该考试类型的成绩，将自动覆盖。

### 4.4 统计分析

**路径：** 侧边栏 → 统计分析

#### 4.4.1 统计概览

- 选择班级和考试类型后点击「查询」
- 统计卡片：学生总数、平均总分、最高总分、及格率、优秀率
- 总分分布柱状图
- 科目趋势折线图（对比各科目历次考试）
- 科目占比饼图
- 能力雷达图（平均分 + 及格率 + 优秀率）
- 总分排名表（可查看各科明细）
- 单科排名表（可选科目）

#### 4.4.2 班级统计

- 选择考试类型后查询
- 概览卡片：学生人数、平均分、及格率
- 班级统计表格（按班级编号排序）
- 柱状图：班级平均分对比
- 折线图：班级及格率/优秀率对比

#### 4.4.3 科目统计

- 选择科目和考试类型后查询
- 概览卡片：最高分、最低分、平均分、及格率、优秀率
- 科目对比柱状图
- 分数分布柱状图（区分 150/100 满分）

### 4.5 班主任管理

**路径：** 侧边栏 → 班主任管理

**分配班主任：**
1. 点击「添加班主任」
2. 选择班级（只显示未分配的班级）
3. 点击「确认」

系统自动：
- 创建用户账号（用户名 = `入学年份 + 班级编号`，如 `2026001`）
- 密码设为 `123456`（首次登录强制改密）
- 自动生成该班级所有学生的成绩汇总数据

**删除班主任：** 列表中选择「删除」，同时删除关联的用户账号。

### 4.6 学科组长管理

**路径：** 侧边栏 → 学科组长管理

**分配学科组长：**
1. 点击「添加学科组长」
2. 选择学科（语文/数学/英语/物理/化学/生物/政治/历史/地理）
3. 填写组长姓名
4. 点击「确认」

系统自动：
- 创建用户账号（用户名 = 学科英文名小写，如 `chinese`）
- 密码设为 `123456`（首次登录强制改密）

> 同一学科只能有一位组长，已分配的学科不会出现在下拉列表中。

### 4.7 教师任课管理

**路径：** 侧边栏 → 教师任课管理

**分配教师任课：**
1. 点击「添加任课分配」
2. 选择学科 + 班级的组合
3. 填写教师姓名
4. 点击「确认」

系统自动创建教师账号（用户名 = `学科英文 + 入学年份 + 班级编号`，如 `Chinese2026001`）。

> 同一教师可分配多个 (学科, 班级) 组合。

### 4.8 账号管理

**路径：** 侧边栏 → 账号管理

包含四个子页面：

#### 4.8.1 学生账号
- 查看所有学生账号：学号、姓名、班级、是否需改密
- 点击「重置密码」：密码重置为 `123456`

#### 4.8.2 班主任账号
- 查看所有班主任账号：用户名、姓名、负责班级
- 点击「重置密码」

#### 4.8.3 学科组长账号
- 查看所有学科组长账号：用户名、学科、姓名
- 点击「重置密码」

#### 4.8.4 教师账号
- 查看所有教师账号：用户名、姓名、任课信息
- 点击「重置密码」

## 10. 常见问题

### Q1: 忘记密码怎么办？
联系管理员。管理员在「账号管理」中找到你的账号，点击「重置密码」将密码重置为 `123456`。下次登录时需修改密码。

### Q2: 无法登录？
- 检查用户名和密码是否正确
- 确认后端服务是否运行（访问 `http://localhost:8000/docs` 测试）
- 首次登录若提示需要改密，请按提示设置新密码

### Q3: 添加班主任时班级下拉为空？
所有班级都已分配班主任。可先删除某位班主任再重新分配。

### Q4: 添加学科组长时学科下拉为空？
所有学科都已分配组长。同一学科只能有一位组长。

### Q5: 班主任/学科组长能看到其他班的数据吗？
不能。班主任只能看到自己负责班级的数据；学科组长只能看到自己负责学科的全年级数据。都是只读权限。

### Q6: 学生能看到其他同学的成绩吗？
不能。学生只能看到自己的成绩。

### Q7: 数据保存在哪里？
所有数据保存在项目根目录的 `data/grades.db` 文件中（SQLite 数据库）。**请定期备份此文件。**

### Q8: 如何备份数据？
将 `data/grades.db` 复制到安全位置即可。恢复时将备份文件覆盖回原位置。

### Q9: 如何修改端口？
- 后端：修改启动命令中的 `--port` 参数
- 前端：编辑 `frontend/vite.config.ts` 中的 `server.port`

### Q10: 启动报错"端口被占用"？
关闭占用该端口的程序，或修改端口。Windows 上可使用 `netstat -ano | findstr :8000` 查看占用进程 PID，在任务管理器中结束该进程。

### Q11: 如何更新系统？
```bash
# 1. 备份数据库
cp data/grades.db data/grades.db.bak

# 2. 拉取最新代码
git pull

# 3. 更新依赖
pip install -r requirements.txt
cd frontend && npm install && cd ..

# 4. 重启服务（关闭旧窗口，重新运行 start.bat）
```
